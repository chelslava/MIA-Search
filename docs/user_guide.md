# Руководство пользователя MIA Search (v0.1.2)

## 1. Установка и запуск

### Вариант A: готовый установщик Windows
1. Скачайте `.exe` из GitHub Release.
2. Запустите установщик и завершите установку.
3. Откройте `MIA Search` из меню Пуск.

### Вариант B: запуск из исходников
```bash
npm install
npm run dev
cd src-tauri
cargo tauri dev
```

## 2. Быстрый старт поиска
1. Введите запрос в верхнем поле.
2. Выберите режим кнопками под строкой поиска: `PLAIN`, `WC`, `RE`.
3. Укажите корневые пути в левой панели.
4. Нажмите `Поиск` или включите `Live`.
5. Результаты появляются потоково, без ожидания полного завершения сканирования.

Команды в строку поиска вводить не нужно: режимы и флаги включаются кнопками и переключателями в UI.

## 3. Фильтры и сортировка

Поддерживаются фильтры:
- Тип записи: все/файлы/папки.
- Расширения: одно или несколько.
- Исключение путей: маски подпапок (например `node_modules`, `.git`, `target`).
- Размер: больше/меньше/равно.
- Дата: `created`/`modified`, до/после.

Поддерживаются сортировки:
- По релевантности.
- По имени.
- По размеру.
- По дате изменения.
- По типу.

## 4. Режимы backend: Scan и Index

- По умолчанию используется `Scan`.
- `Index` можно включить в настройках поиска.
- В настройках доступны:
  - переключатель backend;
  - кнопка `Rebuild index`;
  - индикатор `index_status` (`empty/ready`, `entries`, `updated_at`).
- Есть авто-реиндексация по TTL и интервалу проверки.
- Горячая клавиша реиндексации: `Ctrl/Cmd + Shift + R`.

Если индекс пустой, поиск автоматически откатывается на scan-путь.

## 5. Действия с результатами

Для выбранного результата доступны:
- `Open`;
- `Open parent`;
- `Reveal in Explorer`;
- `Copy path`;
- `Copy name`.

## 6. История, избранное, профили

- История хранит последние запросы и открытия.
- Избранное хранит закрепленные пути.
- Профили сохраняют набор параметров поиска.
- Все данные сохраняются между запусками приложения.

## 7. Советы по производительности

- Для больших директорий ограничивайте `depth` и `limit`.
- Если идет live-поиск, новый запрос автоматически отменяет предыдущий.
- Для ускорения повторных поисков включайте индексный backend.

## 8. Диагностика ошибок поиска

Статус-бар показывает унифицированные ошибки поиска:
- `SEARCH_INVALID_QUERY` — ошибка в шаблоне запроса (regex/wildcard).
- `SEARCH_STATE_ERROR` — внутренняя ошибка состояния search-сессии.
- `SEARCH_EXECUTION_ERROR` — ошибка выполнения поиска в backend.

Практика: при повторяющейся ошибке сохраните текст статуса и приложите его в issue.
