# Roadmap (актуально на 25 марта 2026)

Документ обновлён по фактическому состоянию репозитория на **25.03.2026**.

## 1. Текущее состояние проекта

### Что уже реализовано
- Desktop-приложение на `Tauri 2 + React + TypeScript`.
- Поисковое ядро на Rust (`rust_search`) с потоковой выдачей и отменой.
- Multi-root поиск, depth/limit, strict/ignore_case/include_hidden.
- Режимы запроса: `plain`, `wildcard`, `regex` + inline-команды в строке поиска.
- Фильтры (тип, расширения, размер, дата), сортировки, dedupe.
- Действия с результатами: open/open parent/reveal/copy.
- Stores: settings/history/favorites/profiles с persistence.
- Компактный UI (левое дерево дисков/папок, верхняя панель, статус-бар, детали).
- Темы и RU/EN локализация интерфейса.
- Тесты backend и frontend smoke.

### Качество и тесты
- `cargo test`: зелёный.
- `cargo llvm-cov --summary-only`: **TOTAL Regions Cover 81.14%**.
- `npm run check`, `npm run test`, `npm run build`: зелёные в локальном окружении.

## 2. Целевой MVP

MVP считается готовым, когда:
1. Поиск стабилен на больших деревьях и корректно работает для `plain/wildcard/regex`.
2. Windows release-сборка проходит и подтверждена smoke-проверкой.
3. Документация пользователя и релизный чек-лист синхронизированы.
4. Не остаётся P0-блокеров по UI/поиску/событиям.

## 3. План выхода в MVP (финальный коридор)

## Этап M1: Стабилизация (25-27 марта 2026)
- [x] Исправить wildcard-поиск по нескольким root при лимитах.

## 4. Post-MVP roadmap (после 2 апреля 2026)

## R1: Производительность и UX (апрель 2026)
- Расширенные метрики (TTFR, scanned paths/sec, access errors).
- Оптимизация сканирования больших root и отзывчивости UI.
- Улучшение виртуализации и компактных режимов представления.

## R2: Кроссплатформенная эксплуатация (апрель-май 2026)
- Полноценные smoke прогоны Linux/macOS.
- Кроссплатформенные CI-сборки и артефакты.
- Платформенные корректировки `open/reveal/clipboard`.

## R3: Продуктовые расширения (май 2026)
- Расширенная командная палитра и пользовательские команды.
- Гибкие профили и экспорт/импорт конфигурации.
- Дальнейшая декомпозиция и упрощение frontend-кода.

## 5. Риски и контроль
- Риск: различия поведения файловых API на Windows/Linux/macOS.
- Риск: деградация скорости на больших сетевых/внешних дисках.
- Риск: регрессии при активной доработке UI.

Меры контроля:
- держать `cargo test` и `npm run check` обязательными перед коммитом;
- на каждый фикс поиска добавлять regression-тест;
- релизные изменения сверять по smoke-checklist.
