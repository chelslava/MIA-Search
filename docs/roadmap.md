# Дорожная карта MIA Search (актуально на 26 марта 2026)

## 1. Текущее состояние

Что уже сделано:
- Реализован рабочий desktop-поиск на `Tauri 2 + React + TypeScript + Rust`.
- Поддерживаются режимы `plain`, `wildcard`, `regex`.
- Работают потоковая выдача результатов и отмена поиска.
- Реализована двухфазная модель: сначала легкие результаты, затем догрузка метаданных.
- Есть два backend-режима: `Scan` и `Index` (с fallback на scan, если индекс пуст).
- Реализованы команды индексатора: `index_rebuild`, `index_status`.
- Добавлены фильтры, сортировки, история, избранное, профили и действия по файлу.
- Настроены CI/workflow и release-процессы для GitHub.

Технический уровень качества:
- backend тесты проходят;
- frontend проверки (`check/test/build`) проходят;
- покрытие backend держится на уровне 80%+.

## 2. Цели версии 0.1.1

Приоритеты версии:
1. Стабильность поиска на больших наборах файлов.
2. Надежная индексация и понятный UX управления индексом.
3. Стабильный CI перед публикацией в GitHub.
4. Актуальная документация для пользователя и команды релиза.

## 3. Ближайший план (1-2 спринта)

### Блок A. Надежность и производительность
- Убрать оставшиеся подвисания UI при активном стриме результатов.
- Усилить регрессионные тесты для поиска и индексного fallback.
- Добавить контрольные метрики производительности в smoke-проверки.

### Блок B. Индексация
- Довести авто-реиндексацию до предсказуемого поведения на старте/по TTL.
- Добавить защиту от параллельных rebuild-задач.
- Улучшить диагностику `index_status` (ошибки и рекомендации в UI).

### Блок C. Релизный контур
- Держать зеленым `ci.yml` на `windows/ubuntu/macos`.
- Поддерживать release notes по схеме `docs/github_release_<tag>.md`.
- Зафиксировать единый ритуал Go/No-Go перед тегом.

## 5. Критерии готовности релиза

Релиз считается готовым, если:
1. Нет открытых `P0`.
2. CI зеленый на целевых платформах.
3. Пройдены smoke-проверки из `docs/release_smoke_checklist.md`.
4. Обновлены `README`, `docs/user_guide.md` и release notes под текущий тег.
